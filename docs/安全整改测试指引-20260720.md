# shotcat 安全整改本机测试指引手册（2026-07-20）

> **给谁看**：负责本机验收的普通用户（不需要懂代码）。
> **环境**：Windows 11，终端用 **cmd（命令提示符）** 或 **PowerShell**。本手册命令默认在 **cmd** 里执行；个别用 PowerShell 的会标注 `[PowerShell]`。
> **测什么**：把安全整改（阶段一~四）在本机跑一遍，逐项确认是否通过。整改内容见 `docs/安全整改规划-20260719.md`。

## 怎么算测过（总判定）

验收分三类，按顺序做：

| 类别 | 内容 | 是否必做 | 大致耗时 |
|---|---|---|---|
| **A. 自动化测试** | 一条命令跑 354 个后端测试，覆盖全部整改项 | ✅ 必做 | ~1 分钟 |
| **B. 端到端验收** | 一条命令跑真实环境检查（登录/隔离/限流等），看 PASS/FAIL | ✅ 必做 | ~3 分钟 |
| **C. 部署层验证** | 用 server.bat 验证对外网关的门禁/加密 | ⬜ 对外上线前做 | ~15 分钟 |

**结论标准：A 显示 `354 passed` + B 显示 `✅ 应用层端到端验收通过` = 应用层验收通过；C 全部通过 = 具备对外上线条件。**

> 约定：项目根目录 = `E:\Github\maoShu\shotcat`，后端目录 = `E:\Github\maoShu\shotcat\app\backend`。
> cmd 里进后端目录用：`cd /d E:\Github\maoShu\shotcat\app\backend`（`/d` 能跨盘切换）。

## 0. 只做一次的准备

2. （可选，防中文乱码）在每个 cmd 窗口先执行一次：

## 1. 起测试环境（每次测试前都要做）

### 1.1 ⚠️ 第一步：起“保活”窗口（最重要，不做后面全连不上）

这台机器的 WSL 有个毛病：**没人occupying它时会自动关掉，数据库容器跟着停，然后什么都连不上**。所以先起一个“保活”窗口挂着：

- 新开一个 **cmd 窗口**，输入下面这行，回车：
  ```
  wsl -- sleep infinity
  ```
- 这个窗口会“卡住没反应”——**这是正常的**，说明保活生效了。**把它最小化，测试全程别关它。**

### 1.2 第二步：一键起服务

- 回到项目根目录 `E:\Github\maoShu\shotcat`，**双击 `run.bat`**（或在 cmd 里输入 `run.bat`）。
- 它会自动起数据库容器 + 后端 + 前端，并**弹出两个终端窗口**（backend、web），稍后浏览器会自动打开 `http://localhost:5273`。
- **判定就绪**：浏览器出现 **登录页**，且后端窗口没有报错刷屏。

> 如果浏览器没自动开，手动访问 `http://localhost:5273`。

### 1.3 第三步：准备两个测试账号（首次或换了数据库才需要）

打开一个**新的 cmd 窗口**（不要用保活窗口），执行：

```
cd /d E:\Github\maoShu\shotcat\app\backend
uv run python -m app.cli.create_user --username alice --password Test1234!
uv run python -m app.cli.create_user --username bob --password Test1234!
```

- 各打印“已创建用户 …”即成功；若提示“用户名已存在”，说明之前建过，直接跳到下一步。
- 然后在浏览器用 **alice / Test1234!** 登录，随便**新建一个项目**（B 类越权检查需要 alice 名下有项目）。登录后可以退出。

## 2. A. 自动化测试（必做）

在后端目录执行：

```
cd /d E:\Github\maoShu\shotcat\app\backend
uv run pytest -q
```

- **预期**：最后一行显示 `354 passed`。
- **判定通过 ✅**：出现 `354 passed`，没有 `failed` 或 `error`。
- 若有失败：记下失败的测试名，截图反馈给开发。

> 这一步用的是内置的临时数据库，**不依赖第 1 步的服务**，其实可以最先跑。

## 3. B. 端到端验收（必做，一条命令）

确认第 1 步的服务已经起好，然后在后端目录执行：

```
cd /d E:\Github\maoShu\shotcat\app\backend
uv run python security_acceptance.py
```

脚本会自动登录 alice/bob 并逐项检查，输出类似：

```
[1/5] 跨用户项目隔离 .......... [PASS]
[2/5] API Key 加密存储 .......... [PASS]
[3/5] 上传大小限制(3MB->413) .......... [PASS]
[4/5] 生成接口限流(第11次429) .......... [PASS]
[5/5] 登录防暴力破解(锁定429) .......... [PASS]
结果: 5 通过, 0 失败, 0 跳过
✅ 应用层端到端验收通过
```

**判定通过 ✅**：最后显示 `0 失败` 且 `✅ 应用层端到端验收通过`。

各项含义与判定：

| 项 | 对应整改 | 通过标准 |
|---|---|---|
| 跨用户项目隔离 | 项目级越权修复 | bob 看不到 alice 的项目、越权访问返回 404、无 token 返回 401 |
| API Key 加密存储 | 2.3 | 数据库里的 api_key 是密文（`gAAAAA` 开头），不是明文 |
| 上传大小限制 | 3.2 | 上传 3MB 图片被拒（413） |
| 生成接口限流 | 3.4 | 同一用户一分钟内第 11 次请求被拦（429） |
| 登录防暴力破解 | 3.1 | 连续错误登录达阈值后被锁（429） |

**关于结果里的特殊情况：**
- 出现 `[SKIP]`：多为“数据库里还没有配过 API Key 的供应商”。可在页面（首页顶栏「LLM 配置」）配一个带 key 的供应商后重跑。
- 出现 `[FAIL]`：把整段输出截图反馈给开发。
- **副作用（正常现象）**：脚本会占用 bob 当前这一分钟的生成次数、并累计几次“登录失败”计数。**反复运行**可能触发限流/登录锁定导致后续 FAIL——这时**把 backend 窗口关掉重新 `run.bat`**（清空内存计数）即可。

## 4. C. 部署层验证（对外上线前做，需管理员）

阶段一（对外网关加固）只有在用 `server.bat` 起的**部署模式**下才生效，dev 模式测不到。上线前用管理员权限跑 `server.bat`，按下表核对（对外地址以 server.bat 输出为准）：

| # | 项 | 怎么测 | 通过标准 |
|---|---|---|---|
| 4.1 | 数据库端口不对外 | 从**另一台电脑**尝试连本机 3306/6379/9000 | 连不上（被拒/超时） |
| 4.2 | 网关登录门禁 | 浏览器打开对外地址 | 弹出账号密码框；口令对才进得去 |
| 4.3 | HTTPS | 看地址栏 | 是 `https://`（纯内网模式会有一次证书告警，属正常） |
| 4.4 | 屏蔽接口文档 | 过门禁后访问 `/docs`、`/redoc`、`/openapi.json` | 都返回 **403** |
| 4.5 | CORS/DEBUG | 请开发核对生效的 `.env` | `CORS_ORIGINS` 是具体域名（无 `*`）、`DEBUG=false` |

## 5. 运维脚本（可选，会改数据，跑前先备份）

在后端目录 `cd /d E:\Github\maoShu\shotcat\app\backend` 执行。

- **项目 owner 回填**（把无主的历史项目划归某用户，可重复跑）：
  ```
  uv run python -m app.cli.backfill_project_owner --owner <用户id>
  ```
- **确保存量 API Key 都已加密**（幂等，可重复跑）：
  ```
  uv run python -m app.cli.migrate_provider_secrets
  ```
- **密钥轮换**（换 `PROVIDER_SECRET_ENC_KEY`；⚠️ 不可重复跑、跑前务必备份数据库；一般只在迁移正式服务器当天做）：
  ```
  uv run python -m app.cli.rekey_provider_secrets --old-key <旧密钥> --new-key <新密钥>
  ```

## 6. 遇到问题怎么办

| 现象 | 原因 | 处理 |
|---|---|---|
| B 脚本报“连不上 backend” / 后端窗口报连不上数据库 | 保活窗口没开或被关了，容器掉线 | 回到 **1.1** 重开保活窗口（`wsl -- sleep infinity`），再 `run.bat` |
| 检查端口通不通 | 想确认数据库端口是否就绪 | `[PowerShell]` 执行：`Test-NetConnection 127.0.0.1 -Port 3306`，`TcpTestSucceeded : True` 即通 |
| B 脚本反复跑后开始 FAIL（限流/锁定） | 占用了配额/触发了登录锁定 | 关掉 backend 窗口，重新 `run.bat`（清空计数），等 1 分钟再跑 |
| alice 登录不上 | 之前测登录锁定把它锁了 15 分钟 | 等 15 分钟，或关掉 backend 窗口重开 |
| `uv` / `pnpm` 找不到 | 依赖没装 | 先在项目根跑 `install.bat` |
| 端口 8000/5273 被占用 | 之前的窗口没关 | 关掉旧的 backend/web 窗口再 `run.bat` |
| 中文显示乱码 | 控制台编码 | 先执行 `chcp 65001` 再跑命令 |
