# M4 执行计划与任务指派

> 配套方案：[对象存储访问加固方案-20260721.md](对象存储访问加固方案-20260721.md)
> 本文只讲**怎么执行 / 谁来做 / 用哪个模型 / 怎么省额度**；what/why 见方案文档。

## 0. 总则（模型与 subagent 分配、额度控制）

- **主会话模型**：**Sonnet 5**（日常编码主力）。M4 是常规编码（改 3–4 个文件 + 一个 CLI），非架构/疑难，不需要 Opus。新会话请先 `/model` 切 Sonnet 5，攻坚才临时升 Opus、完事切回。
- **subagent 指派原则**（依据 `~/.claude/rules/model-selection.md`）：
  - M4 改动面小且集中，**默认主会话串行直接做**，不下放——下放反而多花 token。
  - 唯一建议下放的：**只读定位**「全仓所有 `public-read` / `storage.upload_file` 调用点」→ 用 **Explore（haiku）** 一次性摸清，避免主会话反复 grep 污染上下文。也可主会话一条 Grep 搞定则不必起 subagent。
  - **禁止**为省事并行起多个 subagent（本机走会截断的公司代理，并行翻倍截断率与额度）。
- **额度控制**：主会话固定 Sonnet 不中途换模型（换模型使 prompt cache 全失效）；长会话每约 50 次工具调用考虑 `/compact`；只读探索一律 haiku。
- **代码审计**：M4 涉及访问控制（安全敏感），**三阶段全部完成后**跑一次 `security-review` skill（而非每阶段都跑，省额度）。

## 1. M4-1 上传加固（先做）

**范围**：`services/studio/files.py` `upload_file`——key 改 `files/{uuid}{ext}`、去 `ACL=public-read`、`thumbnail` 存代理 URL；清 `utils/files.py` 及 worker 上传点的 public-read；`storage.py` 上传不默认公开。

**步骤（TDD）**：
1. （可选，haiku Explore）定位全仓 `public-read` / `upload_file` 调用点清单。
2. 先写红：`tests/test_studio_files_service.py`（或新增）断言——上传后 `FileItem.storage_key` 形如 `files/<uuid>.<ext>` 不含原始文件名、上传未传 `ACL=public-read`、`FileItem.thumbnail == /api/v1/studio/files/{id}/download`。
3. 实现至绿。
4. 全量回归：`./.venv/Scripts/python.exe -m pytest -q`。

**指派/模型**：主会话 Sonnet 直接做。定位步骤可 haiku Explore。
**提交**：`feat: M4-1 上传加固（UUID key + 去 public-read + thumbnail 代理 URL）`

## 2. M4-2 存量 thumbnail 回填

**范围**：新增幂等 CLI `app/cli/backfill_file_thumbnails.py`，把 `FileItem.thumbnail` 从 bucket URL 重写为代理 URL。仿现有 `app/cli/backfill_*.py` 结构。

**步骤（TDD）**：先写 CLI 单测（内存 SQLite，seed 若干 FileItem 旧 URL → 回填后为代理路径 + 重复执行幂等）→ 实现 → 全量绿。
**指派/模型**：主会话 Sonnet。
**提交**：`feat: M4-2 存量 FileItem.thumbnail 回填代理 URL 幂等 CLI`

## 3. M4-3 bucket 私有化落地

**范围**：`storage.py` `init_storage` 不建公开策略；`_build_public_url`/`s3_public_base_url` 收窄注释为 dev-only；方案文档补运维关 bucket 匿名读步骤。
**步骤**：改代码 + 文档；全量绿；（本机无 RustFS，私有化 403 验收留运维在真实环境执行，代码侧以"不再产生公开 URL/ACL"为准）。
**指派/模型**：主会话 Sonnet。
**提交**：`feat: M4-3 对象存储私有化（停建公开策略 + public URL 收窄 dev-only）`

## 4. 收尾

- 三阶段完成后跑 `security-review` skill 一次。
- 更新方案文档 §5 标注 M4-1/2/3 落地情况（docs commit）。
- **上线运维序（不可颠倒）**：部署 M4-1 → 跑 M4-2 回填 → 运维关 bucket 匿名读。
