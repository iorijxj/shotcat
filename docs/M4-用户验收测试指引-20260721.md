## 一、写在前面

这次改造做的事情，用大白话说就是：

之前你上传/生成的图片、视频，存在服务器上时是"只要知道网址谁都能直接打开"的，存在被人绕过登录偷看/盗链的风险。这次改造之后，素材文件只能通过产品内部的正常操作访问，"裸网址"直连的路会被堵掉。

**这次测试不是要你去找安全漏洞，而是要确认一件事：改造之后，你平时用的功能（生成图片、查看素材、下载导出）一切照旧，没有被这次改动搞坏。** 你在界面上应该看不出任何变化——如果你看出变化了（比如图裂了、按钮点了没反应），那就是问题，请记录下来。

## 二、测试须知

- **环境**：本机开发环境（`run.bat` 启动的 backend + web），不涉及正式对外环境
- **前置条件**：本机服务已经启动，能正常打开产品页面（一般是 `http://localhost:5273`）
- **账号**：用你日常测试的账号登录即可，不需要特殊权限
- **预计用时**：10–15 分钟
- **重要提醒**：产品里目前没有"手动上传文件"的按钮，素材图片/视频都是你点「生成」之后由 AI 自动生成入库的（比如角色定妆图、镜头首帧图）。所以下面的测试用例都是围绕"生成 → 查看 → 下载/导出"这条链路来的，不涉及手动上传。
