# shotcat 公众化开发规划(总纲)

> 状态:规划定稿(2026-07-20),**后续严格按本规划执行**
> 触发:新增需求——部署到外部服务器、面向公众的产品、同时服务国内外用户
> 本文件是总纲,负责编排所有工作流的顺序/依赖/优先级;各工作流细节见对应子文档。

## 0. 背景与既有基线

shotcat 由"局域网内部工具"升级为**面向公众、外部服务器部署、国内外同时访问**的产品。

**已完成基线**(安全整改四阶段,354 测试全绿):轻量 JWT 鉴权、登录锁定、上传限额(2M 图/5M 视频)、SSRF 三段防护、生成限流(10 次/分)、Provider 归属校验、密钥加密(`PROVIDER_SECRET_ENC_KEY`)+ rekey 轮换工具链、任务链接多态归属校验。

**核心约束(定位不变)**:
- `app/` 是平台底座,尽量低侵入(优先加模块/字段/中间层);多租户维度是被接受的必要例外。
- 角色/场景名称全程原样保留;实体 ID 用 char_001/scene_001/prop_001。
- 不重复造底座已有能力。不可变、函数<50 行、文件<800 行。

**两条待定的外部约束(阻塞对应工作流,不阻塞其余)**:
- **服务器位置未定**(境内备案 / 香港 / 境外 / 双节点;GFW 使"单机同时服务国内外"物理受限)→ 阻塞 M7。
- **认证方式未定**(OIDC 或其他第三方网关注入身份)→ 阻塞 M6;但身份接缝已在 M2 收敛好。

## 2. 推荐执行顺序(严格遵循)

```
阶段一  M1 公网安全加固          ← 独立、低风险、先清,为公网暴露打底
阶段二  M2 多租户硬隔离 P0→P5    ← 骨干,数据模型基石,M3 依赖它
阶段三  M3 配额 + 计费风控        ← 挂在租户上
阶段四  M4 对象存储签名 URL       ← 存储访问改造
── 以下阻塞,解除后再排 ──
        M6 外部认证接入          ← 认证方式定了再做(M2 已留接缝,只改 get_current_user)
        M7 外部服务器部署        ← 服务器位置定了、功能就绪后
可并行  M5 i18n + 本地化         ← 纯前端,任意阶段插空推进
```

**排序理由**:M1 项独立且成本低(小时级),先清可快速降低公网基础风险;M2 是数据模型基石,M3 配额必须按租户计量故排其后;M4/M5 相互独立;M6/M7 被外部决策阻塞,接缝已预留,不占关键路径。

## 3. M1 公网安全加固(快赢清单)

从局域网暴露到公网,把防护从"网关 + 仅局域网"下沉到应用层。各项独立,可逐条验收。

| 任务 | 说明 | 验收 |
|------|------|------|
| **弱口令启动校验** | `.env` 现有 `MYSQL_ROOT_PASSWORD=change-me-root`、`MYSQL_PASSWORD=change-me`、RustFS `rustfsadmin/rustfsadmin` 等默认值,上公网即裸奔。启动时检测到默认/弱口令直接拒绝启动 | 用默认口令启动被拒并给出明确提示;改强口令后正常 |
| **应用层限流扩展** | stock Caddy 无 rate_limit 模块,限流必须在后端。现仅生成接口 10 次/分,扩展到登录/(将来)注册/上传等高危接口 | 超阈值返回 429;各接口阈值可配 |
| **CORS 可配置** | `CORS_ORIGINS` 现写死 localhost;改为环境变量驱动,支持真实域名 | 配置域名后跨域正常;未列域名被拒 |
| **上传/SSRF/锁定阈值公网重审** | 上传限额(2M/5M 对真实短剧视频偏小)、SSRF 白/黑名单、登录锁定阈值按公网强度重估 | 阈值有据可依并文档化 |

### 3.1 M1 阈值重审结论(2026-07-20)

四项均已落地并逐项独立提交,354→371 测试全绿。阈值重审的取舍如下:

- **CORS**:本已环境变量驱动,加启动期 fail-fast——`allow_credentials=True` 下 `CORS_ORIGINS` 含通配 `*` 直接拒绝启动,强制逐个列真实前端域名。
- **弱口令启动校验**:安全默认 `ALLOW_WEAK_SECRETS=false`,检测到 `change-me`/`rustfsadmin` 等仓库默认口令(DB/S3/Redis/JWT/Provider 加密密钥)即拒绝启动;仅 LAN 防火墙内开发显式置 true 豁免。
- **登录限流**:新增 `login_rate_limit_per_minute`(默认 60/分,按来源 IP),挡请求洪泛;与既有按失败次数锁定(`login_max_failures_*`)互补。经 Caddy 反代 IP 退化为网关地址,近似全局洪泛上限。
- **上传限额**:2M/5M → **10M 图 / 200M 视频**,覆盖真实短剧参考图与样片;保留硬上限防单请求撑爆存储,总量滥用留给 M3 按租户配额。
- **SSRF**:`ssrf_guard` 采用「只放行 `is_global` 公网 IP」的默认拒绝策略,已覆盖私有网段/回环/链路本地(含云元数据 169.254)/保留/CGNAT,公网强度足够,`ssrf_allow_private_targets=false` 维持不变,不再维护易漏黑名单。
- **登录锁定阈值**:username 维度(精确、不受反代退化影响)保持 5 次/15 分,足挡定向撞库又不误锁;IP 维度因反代退化保持 30 宽松兜底,收紧只会连坐同网关用户,故不动。

## 4. M2 多租户硬隔离(骨干)

详见 [多租户改造方案-20260720.md](多租户改造方案-20260720.md)。要点:引入 `tenants`+`tenant_memberships`+`users.default_tenant_id`;只在聚合根加 `tenant_id`;用 SQLAlchemy `with_loader_criteria` 做 session 级默认安全过滤;身份经新增 `get_current_tenant` 注入、`get_current_user` 不动;引入 Alembic 承接破坏性 DDL;分 P0–P5 六阶段,每阶段 354 测试全绿为基线。

已定决策:租户维度同时支持 B2C(1 人租户)+B2B;LLM 配置每租户各配一套;删除 `project_id` 为空的公共资产并转 NOT NULL;本期不做 `/register`(CLI 建号);role 只落 owner/member、不做租户切换。

## 5. M3 配额 + 计费风控(待 M2)

**目标**:面向公众后,任意用户可刷昂贵的 LLM 图片/视频生成 → 账单爆炸 + 队列挤占。必须按租户配额挡住滥用。

**范围(待细化为子文档)**:per-tenant 生成配额(日/月额度)、用量计量与展示、超额拒绝/降级、是否收费(收费才接支付,先做硬配额)。依赖 M2 的租户维度做计量主体。

## 6. M4 对象存储签名 URL(独立)

**目标**:RustFS 资产不直接暴露公网。改为后端签发短期签名 URL 访问,避免枚举/盗链。**范围(待细化)**:上传/下载走签名、过期策略、与租户隔离对齐(只能签自己租户的资产)。

## 7. M5 i18n + 本地化(可并行)

**目标**:直接由"国内外用户"派生,与服务器位置/认证无关,纯前端起步。**范围(待细化)**:`web/` 界面与报错中英双语、语言切换、时间按用户时区显示。

## 8. M6 外部认证接入(阻塞:认证方式待定)

**目标**:接入 OIDC 或其他第三方网关认证,替换占位的 JWT 登录。**接缝已就绪**(M2 后):身份收敛在 `get_current_user`,换认证本质只改该函数体 + 前端 `auth.ts`/`api.ts`;若外部身份带 tenant claim,`get_current_tenant` 解析分支相应调整。解除阻塞条件:认证方式确定。

## 9. M7 外部服务器部署(阻塞:服务器位置待定)

**目标**:从"WSL 单机局域网"迁到真·Linux 外部服务器。现有 WSL 专属机制(keepalive、vmIdleTimeout、NAT portproxy、Windows 上跑 Caddy、install/server/test/run.bat)大部分作废,需重写为 Linux compose + systemd/CI;镜像走 CI→私有 registry→服务器 pull。**可复用**:Caddy 的 `PUBLIC_DOMAIN` 自动 HTTPS 分支。解除阻塞条件:服务器位置(境内备案/香港/境外/双节点)确定;GFW 决定是否需双节点+GeoDNS(引出跨境数据同步难题)。
